# クレデンシャルバンドル仕様（案）

暗号化済みバンドルは URL に含めやすいよう Base64URL (パディングなし) で運ぶことを前提とし、1 バイトでも短くする方針を取る。

## 暗号化済みバンドルのバイト列構造
- 先頭 8 バイト: 鍵識別子。公開鍵の SHA-256 ハッシュ値の先頭 8 バイトを採用。
- 9 バイト目以降: 暗号化されたクレデンシャルデータ本体。

## 暗号化前クレデンシャルデータ（共通部）
- 0–1 バイト目 (計 2 バイト): データ型。上位 1 バイトを「型カテゴリ」、下位 1 バイトを「マイナーバージョン」とし、マイナー変更は下位バイトの増分で表現する（初版はいずれも 0x00）。
- 2–7 バイト目 (計 6 バイト): expirationTime。VAPID サーバーの期限・キーペアの有効期限・発行時指定の有効期限のうち「最短」を採用し、無期限は認めない。
- 8–9 バイト目 (計 2 バイト): nonce。暗号学的に安全な乱数。

## 型カテゴリとマイナーバージョン
マイナーバージョンはカテゴリごとに下位 1 バイトで管理し、互換性を保ちながら拡張する。

- ### カテゴリ 0x01（非圧縮・素の値 / 汎用）
  - **0x00**: p256dh長 (2 バイト) + p256dh本体(Base64URLデーコード済み) / auth長 (2 バイト) + auth本体(Base64URLデーコード済み) / endpoint 長 (2 バイト) + endpoint本体
  - **0x01**: p256dhのECDH公開鍵の圧縮形式（33 バイト固定） / auth長 (1 バイト) + auth本体(Base64URLデーコード済み) / endpoint長 (2 バイト) + endpoint 本体。

- ### カテゴリ 0x11（FCM 系圧縮 / Chrome/Chromium）
  - 0x0101をベースにendpointの表現を以下に置き換え
  - **0x00**: Chromium 向けサブドメイン付き。`https://{subdomain}.google.com/fcm/send/` をプレフィックスとし、それ以降の`:`前後をBase64URLデコード済みデータをトークン１，トークン２とする。長さ（1バイト）+本文 の表現をサブドメイン・トークン１・トークン２の順で設定する
  - **0x01**: Chrome 向け（サブドメインなし）。`https://fcm.googleapis.com/fcm/send/` をプレフィックスとし、0x00のサブドメイン分を省略

- ### カテゴリ 0x21（Firefox / Autopush）
  - 0x0101をベースにendpointの表現を以下に置き換え
  - **0x00**: `https://push.services.mozilla.com/wpush/v2/` をプレフィックスとし、それ以降をトークンとしてBase64URLデコードしたバイナリとして長さ（1バイト）+本文で表現
  - **0x01**: `https://updates.push.services.mozilla.com/wpush/v2/` をプレフィックスとして0x01と同様

- ### カテゴリ 0x31（Safari）
  - 0x0101をベースにendpointの表現を以下に置き換え
  - **0x00**: Base64URL トークン。`https://webpush.apple.com/api/push/` をプレフィックスとし、トークンを Base64URL デコードした長さ 2 バイト + 本体で格納。
  - **0x01**: 生文字列トークン（フォールバック）。マイナーバージョン0x00と同様だが、トークンを UTF-8 生文字列として長さ 2 バイト + 本体で格納


## 短縮の工夫（今後の検討）
- 既知のエンドポイントプレフィックスを辞書化して差分のみを格納するカテゴリを追加する。
- エンドポイントを Huffman/ブロック圧縮したカテゴリ (型カテゴリ 0x00 など) を用意し、長い URL を短縮する。
- nonce や expirationTime など固定長領域はビットパックの余地があるが、実装・相互運用優先でまずは上記固定長を採用。
